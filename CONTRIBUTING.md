# Contributing to MagnetarEidolon

## Development Setup
1. Clone the repository.
2. Create and activate a virtual environment (if code components are added).
3. Review `RULES.md`, `PLAN.md`, and `WIP_GUIDELINES.md` before starting.
4. Instantiate or update project YAML in `projects/` for your workstream.

## Contribution Workflow
1. Create a branch using canonical naming.
2. Link your work to task IDs in `PLAN.md`.
3. Update documentation and `BITACORA.md` with state changes and decisions.
4. Run validation checks.
5. Open PR with summary, risks, and testing evidence.

## Pull Request Requirements
- References affected tasks.
- Includes relevant `BITACORA.md` entries.
- Keeps `STATUS.md` and `PLAN.md` aligned.
- Documents blockers and mitigation where relevant.
